# PROJECT_BRIEF.md

## 프로젝트 개요

`ixissage`는 인공지능 과목 수업 프로젝트를 위한 Android 데모 앱이다.

목표는 문자 메시지 본문을 AI 모델에 입력하고, 모델이 출력한 정상/스미싱 확률을 앱 화면에 보여주는 것이다. 상용 보안 제품을 만드는 것이 목표가 아니며, Google Play Store 배포도 목표가 아니다. 이 프로젝트는 AI가 문자 스미싱 탐지 문제에서 어떤 역할을 할 수 있는지 학습하고 시연하는 데 초점을 둔다.

LG U+의 익시오 앱에서 영감을 받았지만, ixissage는 실제 통신사 서비스가 아니라 교육용 프로토타입이다.

## 핵심 문제 정의

이 프로젝트는 생성형 AI 문제가 아니다.

문자 본문을 입력으로 받고, 정상 또는 스미싱 라벨을 출력하는 supervised text classification 문제다.

- 입력 `x`: 한국어 문자 메시지 본문
- 출력 `y`: 정상 또는 스미싱 라벨
- 모델 출력: 정상 확률, 스미싱 확률
- 앱 표시: 모델 확률을 바탕으로 `정상`, `주의`, `스팸 경고` 같은 배지 표시

앱은 메시지 내용을 직접 해석해서 위험 여부를 결정하지 않는다. 위험 판단은 반드시 AI 모델의 softmax probability 또는 sigmoid probability에서 나온다.

## 가장 중요한 원칙

이 프로젝트는 AI 모델 중심 프로젝트다.

룰 기반 탐지는 구현하지 않는다. 다음과 같은 구현은 금지한다.

- `"택배"`라는 단어가 있으면 위험하다고 판단하는 코드
- `"미납"`이라는 단어가 있으면 위험하다고 판단하는 코드
- `"URL"` 또는 `http`가 있으면 위험하다고 판단하는 코드
- `"환급"`, `"본인인증"` 같은 키워드를 if문으로 검사하는 코드
- `RuleBasedAnalyzer`, `KeywordDetector`, `UrlRiskRuleEngine` 같은 구조

허용되는 것은 AI 모델 입력을 위한 전처리뿐이다.

- 공백 정리
- 너무 긴 문장 자르기
- 토크나이징
- 패딩
- 모델이 요구하는 정규화

단, 전처리가 위험 판단 로직이 되면 안 된다. 예를 들어 URL을 특수 토큰으로 바꾸는 것은 모델 입력 정규화로 검토할 수 있지만, URL이 있다는 이유로 위험 배지를 붙이는 것은 금지다.

## 사용자 경험 목표

처음 앱을 실행하면 사용자는 문자 메시지 목록을 본다.

각 메시지에는 다음 정보가 표시된다.

- 보낸 사람 또는 샘플 발신자
- 문자 본문 일부
- 모델이 계산한 스미싱 확률
- `정상`, `주의`, `스팸 경고` 배지

메시지 상세 화면에서는 전체 본문과 모델 분류 결과를 확인할 수 있다.

초기 버전에서는 실제 SMS 권한을 사용하지 않는다. `sample_messages.json` 또는 데이터셋 일부를 사용해 데모한다. 실제 `READ_SMS` 권한은 후반부 선택 기능으로만 다룬다.

## 데이터셋

첫 번째 목표 데이터셋은 Hugging Face의 `meal-bbang/Korean_message`다.

출처:

- https://huggingface.co/datasets/meal-bbang/Korean_message

Hugging Face 카드 기준으로 이 데이터셋은 한국어 스팸 메시지 탐지를 위한 텍스트 분류 데이터셋이다. 데이터셋 카드에는 `label 1`이 일상적 문자, `label 2`가 피싱/스미싱 문자라고 설명되어 있다.

원본 CSV 확인 결과 `class` 값은 `1`, `2`, `3`이 존재한다. `label 3`은 데이터셋 카드에 의미가 명시되어 있지 않으므로, 첫 EDA 단계에서 반드시 별도로 확인하고 처리 방침을 기록해야 한다.

## AI 파이프라인 목표

모델 개발은 다음 순서로 진행한다.

1. 데이터셋 다운로드
2. 데이터 구조 확인
3. 라벨 의미와 분포 확인
4. train/dev/test split 생성
5. baseline 모델 학습
6. Transformer 기반 모델 fine-tuning
7. 평가 지표 출력
8. 잘못 분류한 예시 분석
9. `inference.py` 작성
10. Android 데모 앱과 연결 준비

필수 평가 지표는 다음과 같다.

- accuracy
- precision
- recall
- F1-score
- confusion matrix
- false positive 예시
- false negative 예시

스미싱 탐지에서는 recall이 특히 중요하다. 실제 스미싱을 정상으로 놓치는 false negative가 사용자에게 더 큰 위험을 줄 수 있기 때문이다. 다만 이 프로젝트는 교육용 데모이므로, 평가 지표를 통해 trade-off를 설명하는 것이 중요하다.

## Android 앱 범위

Android 앱은 AI 모델 결과를 보여주는 데모 UI다.

초기 앱은 다음 방식으로 만든다.

- Kotlin
- Jetpack Compose
- sample dataset mode
- mock classifier
- 메시지 목록 화면
- 메시지 상세 화면
- 모델 확률과 배지 표시

처음부터 실제 SMS 권한을 다루지 않는다. Android 권한, 개인정보, 기기별 정책 문제 때문에 앱 구현 초반에 `READ_SMS`를 넣으면 AI 프로젝트의 핵심 흐름이 흐려질 수 있다.

실제 SMS 읽기는 후반부 선택 기능이다. 권한 문제로 막히면 sample dataset mode를 유지한다.

## 개인정보 원칙

문자 본문은 외부 서버로 전송하지 않는다.

초기 버전은 로컬 샘플 데이터로 동작한다. 실제 SMS 기능을 추가하더라도 SMS 본문을 서버, 분석 도구, 로그, 외부 API로 보내지 않는다.

특히 다음은 금지한다.

- SMS 본문을 외부 AI API에 전송
- SMS 본문을 Logcat에 그대로 출력
- SMS 본문을 analytics 이벤트로 전송
- 사용자의 실제 문자 데이터를 원격 저장소에 업로드

## 성공 기준

이 프로젝트의 성공 기준은 완벽한 보안 제품을 만드는 것이 아니다.

성공 기준은 다음과 같다.

- 데이터셋을 이해하고 라벨 구조를 설명할 수 있다.
- 룰 기반 탐지 없이 AI 모델로 문자 분류를 수행한다.
- 모델 평가 지표와 confusion matrix를 출력한다.
- 잘못 분류된 예시를 분석한다.
- Android 앱에서 모델 확률 기반 배지를 보여준다.
- sample dataset mode만으로도 데모가 가능하다.
- 실제 SMS 권한이 없어도 프로젝트가 완성될 수 있다.
