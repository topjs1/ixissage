# ixissage

“본 프로젝트는 익시오의 온디바이스 AI 보안 기능에서 영감을 받아, 문자 메시지에 대해 AI 기반 스미싱 탐지를 수행하는 Android 데모 앱을 구현하는 것을 목표로 한다. 기존 룰 기반 필터링이 아니라 Hugging Face 공개 데이터셋으로 학습한 텍스트 분류 모델을 사용해 정상/스미싱 여부를 판단하고, 앱 화면에서 위험도를 시각화한다.”



안드로이드 메시지 앱 접근하기
온디바이스 ai가 메시지 직접 다룰 수 있게 하기
ai가 의심스러운 url을 검사하거나 피싱관련 내용 발견시 위험 알림 띄우기 <스팸 경고>

목적 : 중국만 보더라도 정부에서 전부 관리하는데 메시지나 전화를 중앙 집중식으로 관리하는 것에 대한 반감이 생길 것이다. 이런 수요를 미리 대비하자는 의미.

어떻게 온디바이스 ai가 적은 자원으로 효율적인 검사를 하도록 학습시킬 수 있을까? 
-> 어차피 20대 컴퓨터하는 사람에게 도움을 주는게 주 목적이라기 보다는 취약한 연령대나 취약한 계층의 사람들에게 도움을 주도록 하기 위한 목적이기 때문에, 모든 해킹이나 스미싱 시도를 막는다는건 당연히 말이 되지 않고, 목표도 아님. 단지 헷갈리는 상황일때 참고할 수 있는 지표를 제공하거나 데이터베이스에 저장된 알려진 형태인 경우에는 확실하게 보고 걸러내는게 제1 목표임. 

그래서 기술적으로 필요한 것들을 순서대로 나열해 보자면:

	1. 안드로이드 메시지앱을 접근할 수 있어야함.
		-> 익시오는 보니까 기본 전화앱과 별개로 익시오 전화 앱이 있는것 같던데, 문자메시지도 기본 메시지앱을 굳이 사용할 필요 없이 우리가 직접 새로운 앱(가칭-ixissage)을 만들어서 거기서 문자메시지를 다룰 수 있도록 하는 아이디어가 생각이 났음

	2. ixissage앱이 어떻게 온디바이스 ai를 불러올 수 있는지에 대해서 알아야함.
		-> 또한, 온디바이스 ai가 무엇이고, 어떻게 실행되는지도 알아야함. 어떻게 우리가 온디바이스 ai를 이용할 수 있는지 개념이 아예 없음.

	3. 적당한 지침이 주어진 ai 가 실행되어야함.
		-> ai를 학습시킬 수 있으면 좋다. 근데, 내가 ai 잘 모르기도 하고, 학습한 모델의 크기가 10GB 이상 되어버리면 배보다 배꼽이 큼. 따라서 온디바이스 ai에 대해서 알아보고, 그 지식을 바탕으로 어떻게 특정한 작업을 시킬 수 있을지 고민해야함.

	4. 기본적인 방향성이 잡혔으면, ai가 피싱메시지를 판단하고 경고를 주는 역할을 해야함.
		-> 1차적으로 위험한 메시지인지 <스팸 경고> 같은 표시를 하도록 한다. 그리고 중앙 데이터베이스에서(이건 장기적으로 유플러스같은 통신사에서 관리해야함) 개인 디바이스 방향으로 특정한 룰셋이나 새롭게 나타나고 있는 스팸메세지 형태를 업데이트하고 고지해주도록 하는 기능을 통해서 지속적으로 안전함을 유지해야함.

	5. 결국, 방화벽이 작동하는것처럼 단순한 룰셋으로 필터링하는 것보다는 한발 더 나아가는 역할을 할 수 있어야한다. 

	6. 중국의 정책을 예시로 들자면, 중국의 CBDC 정책과 비슷하게, 정부에서 내 문자메시지 데이터를 관리하는 것에 대한 의심과 불안함이 상승하는 상황에 대비하기 위한 것이 프로젝트의 목적이다. 물론, 통신사를 거치지 않은 메세지 앱들이 많지만, 그래도 가장 기본적인 메세지에서도 검열받지 않고자 하는 욕구가 존재한다고 생각한다.
